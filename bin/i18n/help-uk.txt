Використання: %s [ОПЦІЇ]

Генерує composer require команди з "composer outdated --direct".
Працює з будь-яким PHP проектом (Symfony, Laravel, Yii, CodeIgniter, CakePHP, Laminas, Slim, тощо)

ОПЦІЇ:
    --run                    Автоматично виконує рекомендовані команди
    --release-info           Показує інформацію про реліз (резюме з посиланнями)
    --release-detail         Показує повний журнал змін релізу для кожного пакета (включає --release-info)
    --no-release-info        Пропускає розділ інформації про реліз (поведінка за замовчуванням)
    -v, --verbose            Показує детальний вивід (файли конфігурації, пакети, тощо)
    --debug                  Показує інформацію для налагодження (дуже детально, включає шляхи до файлів, парсинг, тощо)
    -h, --help               Показує це повідомлення довідки

ПРИКЛАДИ:
    %s                                    # Показує рекомендовані команди (без інформації про реліз за замовчуванням)
    %s --run                              # Виконує рекомендовані команди
    %s --release-info                     # Показує резюме інформації про реліз
    %s --release-detail                   # Показує повні журнали змін (включає --release-info)
    %s --verbose                          # Показує детальний вивід з деталями конфігурації
    %s --debug                            # Показує детальну інформацію для налагодження
    %s --run --release-detail             # Виконує і показує повні журнали змін
    %s --verbose --release-info           # Детально з інформацією про реліз
    %s --debug                            # Режим налагодження (дуже детально)

ПІДТРИМКА ФРЕЙМВОРКУ:
    Скрипт автоматично дотримується обмежень версії фреймворку:
    - Symfony: дотримується обмеження "extra.symfony.require"
    - Laravel: дотримується версії laravel/framework major.minor
    - Yii: дотримується версії yiisoft/yii2 major.minor
    - CakePHP: дотримується версії cakephp/cakephp major.minor
    - Laminas: дотримується версій laminas/* major.minor
    - CodeIgniter: дотримується версії codeigniter4/framework major.minor
    - Slim: дотримується версії slim/slim major.minor

ІГНОРОВАНІ ПАКЕТИ:
    Пакети, перелічені в generate-composer-require.yaml або .yml, будуть пропущені.
    Скрипт шукає файли конфігурації в поточній директорії (де знаходиться composer.json).
    Старий формат (generate-composer-require.ignore.txt) все ще підтримується для зворотної сумісності.
    Коментарі, що починаються з #, ігноруються.

ІНФОРМАЦІЯ ПРО РЕЛІЗ:
    За замовчуванням інформація про реліз НЕ показується (API виклики не виконуються).
    Використовуйте --release-info для показу резюме з:
    - Назвою пакета
    - URL релізу
    - URL журналу змін

    Використовуйте --release-detail для перегляду повного журналу змін релізу.
    Використовуйте --no-release-info для явного пропуску інформації про реліз (поведінка за замовчуванням).
