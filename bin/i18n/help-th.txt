การใช้งาน: %s [ตัวเลือก]

สร้างคำสั่ง composer require จาก "composer outdated --direct"
ทำงานกับโปรเจ็กต์ PHP ใดๆ (Symfony, Laravel, Yii, CodeIgniter, CakePHP, Laminas, Slim, ฯลฯ)

ตัวเลือก:
    --run                    รันคำสั่งที่แนะนำโดยอัตโนมัติ
    --release-info           แสดงข้อมูลการเผยแพร่ (สรุปพร้อมลิงก์)
    --release-detail         แสดงบันทึกการเปลี่ยนแปลงการเผยแพร่แบบเต็มสำหรับแต่ละแพ็กเกจ (รวม --release-info)
    --no-release-info        ข้ามส่วนข้อมูลการเผยแพร่ (พฤติกรรมเริ่มต้น)
    -v, --verbose            แสดงผลลัพธ์แบบละเอียด (ไฟล์การตั้งค่า แพ็กเกจ ฯลฯ)
    --debug                  แสดงข้อมูลดีบัก (ละเอียดมาก รวมถึงเส้นทางไฟล์ การแยกวิเคราะห์ ฯลฯ)
    -h, --help               แสดงข้อความช่วยเหลือนี้

ตัวอย่าง:
    %s                                    # แสดงคำสั่งที่แนะนำ (ไม่มีข้อมูลการเผยแพร่ตามค่าเริ่มต้น)
    %s --run                              # รันคำสั่งที่แนะนำ
    %s --release-info                     # แสดงสรุปข้อมูลการเผยแพร่
    %s --release-detail                   # แสดงบันทึกการเปลี่ยนแปลงแบบเต็ม (รวม --release-info)
    %s --verbose                          # แสดงผลลัพธ์แบบละเอียดพร้อมรายละเอียดการตั้งค่า
    %s --debug                            # แสดงข้อมูลดีบักแบบละเอียด
    %s --run --release-detail             # รันและแสดงบันทึกการเปลี่ยนแปลงแบบเต็ม
    %s --verbose --release-info           # แบบละเอียดพร้อมข้อมูลการเผยแพร่
    %s --debug                            # โหมดดีบัก (ละเอียดมาก)

การสนับสนุน Framework:
    สคริปต์จะเคารพข้อจำกัดเวอร์ชันของ framework โดยอัตโนมัติ:
    - Symfony: เคารพข้อจำกัด "extra.symfony.require"
    - Laravel: เคารพเวอร์ชัน laravel/framework major.minor
    - Yii: เคารพเวอร์ชัน yiisoft/yii2 major.minor
    - CakePHP: เคารพเวอร์ชัน cakephp/cakephp major.minor
    - Laminas: เคารพเวอร์ชัน laminas/* major.minor
    - CodeIgniter: เคารพเวอร์ชัน codeigniter4/framework major.minor
    - Slim: เคารพเวอร์ชัน slim/slim major.minor

แพ็กเกจที่ละเว้น:
    แพ็กเกจที่ระบุไว้ใน generate-composer-require.yaml หรือ .yml จะถูกข้าม
    สคริปต์จะค้นหาไฟล์การตั้งค่าในไดเรกทอรีปัจจุบัน (ที่ตั้งของ composer.json)
    รูปแบบเก่า (generate-composer-require.ignore.txt) ยังคงได้รับการสนับสนุนเพื่อความเข้ากันได้ย้อนหลัง
    ความคิดเห็นที่ขึ้นต้นด้วย # จะถูกละเว้น

ข้อมูลการเผยแพร่:
    ตามค่าเริ่มต้น ข้อมูลการเผยแพร่จะไม่แสดง (ไม่มีการเรียก API)
    ใช้ --release-info เพื่อแสดงสรุปพร้อม:
    - ชื่อแพ็กเกจ
    - URL การเผยแพร่
    - URL บันทึกการเปลี่ยนแปลง

    ใช้ --release-detail เพื่อดูบันทึกการเปลี่ยนแปลงการเผยแพร่แบบเต็ม
    ใช้ --no-release-info เพื่อข้ามข้อมูลการเผยแพร่อย่างชัดเจน (พฤติกรรมเริ่มต้น)
